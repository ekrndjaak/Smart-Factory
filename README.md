# 제조업 생산 공정 모니터링 시스템
### 엔진·차체 조립 공정 기반 제조업 IT 설계 프로젝트

---

## 📌 프로젝트 개요

이 프로젝트는 **자동차 엔진·차체 조립 라인에서의 생산직 근무 경험**을 바탕으로,  
조립 공정에서 발생하는 생산 이벤트를 **데이터와 시스템 관점에서 재구성**한 프로젝트입니다.

웹 서비스를 만드는 것이 목적이 아니라,  
**생산 공정 데이터가 어떻게 생성되고 → 저장되고 → 집계되어  
관리자가 활용 가능한 정보로 전환되는지**를 구현하는 데 초점을 두었습니다.

---

## 🎯 프로젝트 목표

- 엔진·차체 조립 공정을 **라인 / 스테이션 단위로 구조화**
- 주·야 교대, 라인 정지, 불량 이벤트를 고려한 **현장 기반 데이터 설계**
- 생산 이벤트를 안정적으로 처리하기 위한 **서버·배치 중심 구조 구현**
- 제조업 IT / MES / 생산 시스템 개발 직무에 적합한 **설계 중심 포트폴리오 구성**

---

## 🏭 가상 생산 공정 구성

### ▪ 생산 라인

- `BODY_LINE_A` : 차체 조립 라인
- `ENG_LINE_B` : 엔진 조립 라인

### ▪ 스테이션(공정 단계)

**차체 조립**
- `BODY_ASSY_1` : 초기 조립
- `BODY_ASSY_2` : 주요 부품 조립
- `BODY_INSPECTION` : 조립 후 검사

**엔진 조립**
- `ENG_ASSY_1` : 주요 부품 조립
- `ENG_ASSY_2` : 부속품 조립
- `ENG_TEST` : 기능 및 품질 검사

각 제품(Unit)은 스테이션을 순차적으로 이동하며  
조립 및 검사 과정에서 **상태 변화 이벤트를 생성**합니다.

---

## 🔄 설계에 반영한 현장 요소

### 교대 근무
- DAY : 08:00 ~ 20:00  
- NIGHT : 20:00 ~ 08:00  

→ 동일한 생산량이라도 **교대별 분석이 가능하도록 설계**

### 라인 정지 이벤트
- 랜덤 발생
- 일정 시간 후 재가동  

→ 생산량 외에 **비가동 시간(Downtime)**을 데이터로 남기기 위함

### 불량 이벤트
- 검사 단계에서 확률적으로 발생
- 야간 교대 시 정지·불량 확률 소폭 증가

> 본 설정은 실제 공장을 그대로 재현하기 위함이 아니라,  
> **왜 이러한 이벤트가 시스템 설계에 반드시 포함되어야 하는지 설명하기 위한 구조적 설계**입니다.

---

## 🧱 시스템 구조 (이벤트 중심)

```text
[가상 생산 이벤트 생성기]
        ↓
[raw_events 테이블 (원본 이벤트)]
        ↓
[집계 및 정합성 처리 로직]
        ↓
[summary 테이블]
        ↓
[조회 API / 콘솔 리포트]
```

- 원본 이벤트와 집계 데이터를 명확히 분리
- 웹 UI는 필수 요소가 아니며, 조회 목적에 한해 최소한으로 사용
- 시스템의 핵심은 **데이터 흐름과 처리 책임의 분리**

---

## 🗃 데이터베이스 설계 (요약)

### raw_events
- 생산 공정에서 발생하는 모든 상태 변화 이벤트 저장
- 이벤트 중복 방지 및 이력 추적 목적

**주요 컬럼**
- timestamp  
- line_id  
- station  
- event_type  
- unit_id  
- cycle_time  
- reason_code  

### summary
- 교대 / 라인 단위로 집계된 생산 데이터

**주요 컬럼**
- date  
- shift (DAY / NIGHT)  
- line_id  
- produced_count  
- defect_count  
- stop_minutes
